# Requirements — qx-53v (plan: calcfast-norm-drift-guard)

Source: `.claude/plans/calcfast-norm-drift-guard/plan.md` (the spec).

## Acceptance criteria

1. Configurable `renormalize: true` option in `run/2` opts.
2. Norm assertion available in test/dev mode.
3. **AMENDED 2026-05-16 (user-approved Option A):** original
   "norm within `1.0e-10` after a 100-gate circuit" is INFEASIBLE in
   `:c64`/float32 (machine floor ≈1.2e-7 even right after
   `Math.normalize/1`; single Hadamard deviates ~6e-8). New AC #3:
   after a 100-gate circuit, `renormalize: 10` keeps |Σ|a|²−1| within
   **`1.0e-6`** AND strictly lower than the same circuit with
   `renormalize: false` (relative-improvement guarantee — renorm
   demonstrably reduces drift). The dev/test guard tolerance is
   `1.0e-6` (`@norm_tolerance`), not `1.0e-10`.
4. No performance regression on short circuits (the default `:off`
   path must be within benchmark noise of the pre-feature baseline).

## Design constraints (from plan + scratchpad)

- `Qx.CalcFast` MUST NOT be modified (scoping correction: the seam is
  `Qx.Simulation` + `Qx.Math` + `Qx.Validation`; the issue title
  "in CalcFast" is the wrong home — CalcFast is stateless per-gate
  `defn` kernels with no run loop/opts).
- Renorm reuses `Qx.Math.normalize/1`; guard reuses
  `Qx.Validation.validate_normalized!/2`. No new hex dependency.
- `:renormalize` resolves to `:off | :measurement | {:every, n}`;
  default `false`/`:off` → zero behaviour/perf change (non-breaking,
  additive — no major version bump; CHANGELOG `[Unreleased]` only,
  no `mix.exs` version bump since release is tag-gated separately).
- Invalid `:renormalize` must raise a typed `Qx.*Error`
  (`Qx.OptionError`), never a raw `ArgumentError`/`FunctionClauseError`
  (Iron Law #7).
- Guard is compile-time gated via `Application.compile_env(:qx,
  :assert_norm, false)` → on in `:test`, off in `:prod`/`:dev` (the
  host sync `Nx.to_number` inside `validate_normalized!/2` must be
  dead code in `:prod`, Iron Law Nx #5).
- Both the non-conditional (`execute_circuit/2`) and conditional
  (`execute_single_shot/2`) paths must honour `:renormalize`.
