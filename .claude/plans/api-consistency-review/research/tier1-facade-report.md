# Tier 1 facade report ‚Äî `Qx` module (60 functions)

Phase 2 scoring of every tier 1 row in `research/inventory.md` against
`spec/api-design-principles.md` ¬ß4‚Äì¬ß8. Source of truth read:
`lib/qx.ex`, `lib/qx/operations.ex`, `lib/qx/patterns.ex`,
`lib/qx/draw.ex`, `lib/qx/draw/tables.ex`, `README.md`.

Functions scored clean and not listed below: the 24 plain gates
(`h/x/y/z/s/sdg/t/rx/ry/rz/phase/u/cx/cy/cz/cp/crx/cry/crz/ccx/cswap/swap/iswap`)
all satisfy the ¬ß5 gate contract ‚Äî controls before targets, angles
last, circuit in/circuit out. `measure/3`'s
`(circuit, qubit, classical_bit) -> circuit` shape is uniform across
the measure family. `steps/2` is lazy and says so. `c_if/4` was checked
specifically: subject first, scalars, function last (same fun-last
convention as `tap_*`), wraps any gate including nested pipelines, and
its `gate_fn :: (circuit -> circuit)` is the only shape that composes
with every builder ‚Äî consistent, no finding (it needs a family row, see
T1-18).

## Findings

- **T1-01** ‚Äî `draw/2`, `draw_counts/2`, `draw_histogram/2`, `draw_bloch/2` ‚Äî ¬ß5 orthogonality + ¬ß6 return shapes ‚Äî **high** ‚Äî All four specs read `VegaLite.t() | String.t()` (lib/qx.ex:1095, 1113, 1144, 1181): the `format: :vega_lite | :svg` option value switches the return *kind*, the exact pattern ¬ß5 forbids and ¬ß9.1 queues for adjudication. Same family, two shapes, chosen at runtime by an option. Fix: one static return per function ‚Äî either a plain-data chart spec with `:svg` as a separate function/renderer, or two functions per chart. Bucket: **breaking-1.0**.
- **T1-02** ‚Äî `draw_state/2` ‚Äî ¬ß6 environment independence + docs accuracy ‚Äî **high** ‚Äî Facade @spec says `String.t()` (lib/qx.ex:1218) and the facade doc offers `:text` (default) / `:html`; the delegate `Draw.state_table` actually defaults to `format: :auto` and sniffs `Code.ensure_loaded?(Kino)` (lib/qx/draw/tables.ex:181), returning `%Kino.Markdown{}` in Livebook ‚Äî the standing violation named in ¬ß6, and the facade doc/spec misdescribe the delegate (`:auto`/`:markdown` undocumented at tier 1). Fix: default `:text`, always return a string, move Livebook richness to a `Kino.Render` protocol impl behind an optional dep; correct the facade doc now. Bucket: doc fix **non-breaking-now**; return-type fix **deprecate-next-minor**.
- **T1-03** ‚Äî `measure/3` vs `measure_z/3` ‚Äî ¬ß4 one obvious way ‚Äî **med** ‚Äî Byte-identical bodies: both call `QuantumCircuit.add_measurement/3` (lib/qx/operations.ex:621-623 vs 638-640). Two taught spellings for one concept; README uses both (`Qx.measure` 14√ó, `Qx.measure_z` 1√ó). The family table lists both, but ¬ß4 still counts it a finding. Fix: pick `measure/3` as the taught name, keep `measure_z/3` as a documented alias until 1.0, then drop one. Bucket: **breaking-1.0** (decision item).
- **T1-04** ‚Äî `barrier/2` vs `barrier_all/2` ‚Äî ¬ß4 duplication + ¬ß6 family semantics ‚Äî **med** ‚Äî `barrier_all(qc, list)` is `barrier(qc, list)` plus range coercion and an empty no-op (lib/qx/patterns.ex:254-262 delegates straight to `Operations.barrier/2`). Also two family breaks: `*_all` means "broadcast over qubits" yet `barrier_all/2` emits *one* spanning instruction, and the pair disagree on empty input ‚Äî `barrier(qc, [])` emits `{:barrier, [], []}` (the exact unproducible-shape ghost from the Iron Law #9 incident) while `barrier_all(qc, [])` no-ops. Fix: make `barrier/2` accept ranges and no-op (or raise) on empty; keep `barrier_all/1` as the whole-circuit form; deprecate `barrier_all/2`. Bucket: **deprecate-next-minor**.
- **T1-05** ‚Äî missing `tdg/2` ‚Äî ¬ß6 `*dg` family completeness ‚Äî **med** ‚Äî `s/sdg` pair exists (lib/qx.ex:459, 477); `t/2` has no dagger. ¬ß6 explicitly demands an answer: add `tdg` or document `phase(-œÄ/4)` as the idiom. QASM interop tips it: `tdg` is a standard OpenQASM gate, so export/import round-trips favour adding it. Fix: add `tdg/2`. Bucket: **non-breaking-now**.
- **T1-06** ‚Äî `run/2` integer overload ‚Äî ¬ß4 two spellings + ¬ß6 argument order ‚Äî **med** ‚Äî `Qx.run(qc, 2048)` and `Qx.run(qc, shots: 2048)` are two spellings of one call; the scalar sits in the options position (lib/qx.ex:926-935, doc line 924 labels it "backward compatible"). No other tier 1 function accepts a bare scalar where opts go. Fix: soft-deprecate the integer clause (doc note now, `@deprecated` next minor, remove at 1.0). Bucket: **deprecate-next-minor**.
- **T1-07** ‚Äî `bell_state/1`, `ghz_state/1`, `superposition/1` ‚Äî ¬ß8 appenders-by-default + ¬ß6 argument order ‚Äî **med** ‚Äî All three are creators with no circuit subject and no appender underneath (delegating to `Patterns.*_circuit/1`), so none can nest inside a larger circuit ‚Äî the ¬ß9.8 tension, confirmed at tier 1. They are the only non-`create_circuit` tier 1 functions that don't take a circuit first. Fix: add appender forms (`bell_pair(circuit, q0, q1)`, `ghz(circuit, qubits)`), reframe the creators as thin teaching wrappers over them, per ¬ß8. Bucket: **breaking-1.0** for any creator removal; appender additions **non-breaking-now**.
- **T1-08** ‚Äî `superposition/1` ‚Äî ¬ß4 composition + ¬ß7 README test ‚Äî **med** ‚Äî Exactly `create_circuit(n) |> h_all()`; the README never calls it (grep of `Qx.*` calls: zero hits) while `bell_state` (3√ó) and `ghz_state` (1√ó) at least appear. ¬ß4: "a variant nobody's notebook calls gets deprecated"; ¬ß7: a tier 1 function the README can't motivate drops a tier. Fix: deprecate, ship the two-step pipeline as the doc example. Bucket: **deprecate-next-minor**.
- **T1-09** ‚Äî `version/0` ‚Äî ¬ß6 naming families ‚Äî **low** ‚Äî Fits no family row, takes no subject, and fails the README test (unused there). It is, however, a widespread Elixir library convention. Fix: add a documented exception (or an "introspection" row) to ¬ß6 rather than touch the code. Bucket: **non-breaking-now**.
- **T1-10** ‚Äî `create_circuit/1,2` ‚Äî ¬ß6 error contract ‚Äî **med** ‚Äî The tier 1 docs *advertise* a raw `FunctionClauseError` escaping the facade ("If `num_qubits`/`num_classical_bits` is not an integer, or `num_classical_bits < 0`", lib/qx.ex:100, 122). ¬ß6: raw `FunctionClauseError` escaping tier 1 is a finding; ¬ß7 error-message test fails too (a clause error names `Qx.QuantumCircuit.new/2` internals, not the fix). Fix: validate in `QuantumCircuit.new` and raise `Qx.QubitCountError` / a typed parameter error for all bad inputs. Bucket: **non-breaking-now** (tightening raise types).
- **T1-11** ‚Äî `tap_state/2`, `tap_probabilities/2` facade docs ‚Äî ¬ß2 build-never-simulates exception ‚Äî **low** ‚Äî ¬ß2 requires the tap docs to say "loudly" that they simulate. The loud text exists only one tier down ("**Important:** This executes all instructions so far", lib/qx/operations.ex:834-835, 889-890); the tier 1 docs a learner reads say only "See `Qx.Operations.tap_state/2` for full documentation" (lib/qx.ex:1329-1331, 1345-1348). `tap_circuit` doesn't simulate, so within the family cost varies invisibly ‚Äî the facade doc must carry the warning. Fix: copy the Important paragraph into the three facade @docs. Bucket: **non-breaking-now**.
- **T1-12** ‚Äî no `to_qasm`/`from_qasm` on the facade ‚Äî ¬ß6 family table + ¬ß7 one-import / tutorial-apology tests ‚Äî **med** ‚Äî The ¬ß6 family table lists `to_qasm`/`from_qasm` as members "today", but they live only on tier 2 `Qx.Export.OpenQASM`. The `Qx` moduledoc itself has to drop a tier to demonstrate export (`Qx.Export.OpenQASM.to_qasm(circuit)`, lib/qx.ex:71) ‚Äî a tutorial apology inside tier 1's own front page. Fix: `defdelegate to_qasm(circuit, opts \\ [])` and `from_qasm/1` + `from_qasm!/1` on `Qx`. Bucket: **non-breaking-now**.
- **T1-13** ‚Äî `draw/2` doc cross-references ‚Äî ¬ß6 docs ‚Äî **low** ‚Äî The doc points to "`histogram/2`" twice (lib/qx.ex:1070-1071, 1092), a function that doesn't exist on `Qx` (it's `Qx.draw_histogram/2`; `histogram/2` is the tier 2 `Qx.Draw` name). Broken autolink at tier 1. Fix: s/histogram\/2/draw_histogram\/2/. Bucket: **non-breaking-now**.
- **T1-14** ‚Äî `draw_state/2` accepts an internal struct ‚Äî ¬ß3 tier discipline ‚Äî **low** ‚Äî Spec is `Nx.Tensor.t() | struct()` and the doc says "(an internal calc-engine register struct also works)" (lib/qx.ex:1191-1192, 1218). A tier 3 type (`Qx.Register`, demoted v0.10, no stability promise) named in a tier 1 signature. Fix: drop `struct()` and the parenthetical from the facade; tier 2 `Qx.Draw.state_table` may keep the escape hatch. Bucket: **deprecate-next-minor**.
- **T1-15** ‚Äî `rx/ry/rz/phase` specs vs `cp/crx/cry/crz/u` ‚Äî ¬ß6 consistency ‚Äî **low** ‚Äî Rotation angles are typed `float()` in one half of the gate family (lib/qx.ex:509, 526, 543, 560) and `number()` in the other (329, 362, 379, 396, 595); all implementations accept integers. Fix: `number()` everywhere. Bucket: **non-breaking-now**.
- **T1-16** ‚Äî missing `## Returns` / `## Raises` sections ‚Äî ¬ß6 docs rule ‚Äî **low** ‚Äî ¬ß6 requires every tier 1 function to carry a `## Returns` section and `## Raises` when it raises. Only `run/2` has `## Returns`; ~50 of 60 lack it. `cz`, `ccx`, `s`, `t`, `rx`, `ry`, `rz`, `phase` raise `Qx.QubitIndexError` but document no `## Raises` (e.g. lib/qx.ex:238-256, 399-417, 495-544). Mechanical sweep. Bucket: **non-breaking-now**.
- **T1-17** ‚Äî `get_state/2`, `get_probabilities/2` raise on measured circuits ‚Äî ¬ß5 boring combinations ‚Äî **low** ‚Äî Both raise `Qx.MeasurementError` for circuits `run/2` and `steps/2` accept (lib/qx.ex:960, 990) ‚Äî a combination that raises, which ¬ß5 calls a finding. The raise is physically motivated (no deterministic post-measurement state) and `steps/2` already serves that need, so this is a documented-exception candidate rather than a code change. Fix: add the exception to the spec file; ensure the error message points at `run/2`/`steps/2`. Bucket: **non-breaking-now**.
- **T1-18** ‚Äî `run`, `steps`, `c_if`, `barrier`, `cx_chain` fit no ¬ß6 family row ‚Äî ¬ß6 naming families ‚Äî **low** ‚Äî Five tier 1 verbs have no row: `run`/`steps` (execution), `c_if` (control flow), `barrier` (marker), `cx_chain` (block appender ‚Äî the shape ¬ß8 says v0.13 will multiply). No code smell found in any of them (`c_if` scored clean above), but ¬ß6 says new functions "join a family or argue for a new row", and the next algorithm-block wave needs rows to join. Fix: add rows (execute, control, `*_chain`/block-appender) to the ¬ß6 table. Bucket: **non-breaking-now**.

## Summary

Scored all 60 tier 1 functions on `Qx`: 18 findings (2 high, 8 med, 8 low).
The two highs are the Draw layer's option-switched return kinds (T1-01) and
`draw_state`'s Kino sniffing behind a wrong facade spec (T1-02). The med tier
is duplication (`measure`/`measure_z`, `barrier`/`barrier_all/2`, `superposition`,
`run`'s integer overload), the creator trio lacking appenders, the missing `tdg`
and QASM facade delegates, and `create_circuit`'s advertised `FunctionClauseError`.
The 24 plain gates, the measure shape, `steps` laziness, and `c_if` all score
clean; buckets: 10 non-breaking-now, 4 deprecate-next-minor, 3 breaking-1.0,
1 split (T1-02).
